# KR Stock Screener PRD

> 한국 전체 종목 주가데이터 기반 주식 스크리닝 도구
> 작성일: 2026-02-27 | 버전: v0.1

---

## 1. 프로젝트 개요

### 1.1 목적
한국 KOSPI/KOSDAQ 전체 종목(약 2,570개)의 주가 데이터를 수집·저장하고, 다양한 기술적 필터를 적용하여 관심 종목을 빠르게 스크리닝하는 로컬 전용 개인 도구를 만들려고 한다.
기존의 파이썬 패키지를 기반으로 웹으로 구현하려고 한다.

### 1.2 핵심 가치
- **빠른 스캐닝**: 필터 적용 후 종목 리스트를 키보드/스크롤로 빠르게 훑으며 차트 확인
- **유연한 필터링**: 시가총액, 기간별 수익률, 기술적 패턴, RS 점수 등 복합 조건 조합
- **섹터 그룹핑**: sectormap 기준 산업별 그룹핑으로 테마/업종 흐름 파악

### 1.3 기술 스택 (예상)

| 구분 | 기술 |
|------|------|
| Backend | Python 3.11+, FastAPI |
| Frontend | React (Vite), TypeScript |
| Chart | TradingView Lightweight Charts |
| DB | SQLite |
| 배포 | 로컬 전용 (localhost) |

## 4. 기능 요구사항

### 4.1 DB 업데이트

| 항목 | 상세 |
|------|------|
| 트리거 | `[DB 업데이트]` 버튼 클릭 |
| 동작 | 전체 종목 일별/주별 OHLCV + 이동평균 + RS 지표 수집·저장 |
| UX | 프로그레스바 + 예상 잔여시간 표시, 백그라운드 실행 |
| 빈도 | 수동 (장 마감 후 1일 1회 권장) |

### 4.2 필터 시스템

**필터 카테고리:**

| 필터 | 입력 방식 | 예시 값 |
|------|-----------|---------|
| 시가총액 | 범위 선택 | 1000억 이상, 5000억 이상, 1조 이상 |
| 기간별 등락률 | 기간 + 임계값 | 1일/1주/1개월/3개월 × 10%/50%/100% 이상 |
| 기술적 패턴 1 | 조건 빌더 | 종가 ≤ 10일 이평선 × 1.05 |
| 기술적 패턴 2 | 조건 빌더 + AND/OR | 10일/20일/50일 이평선 이격 ≤ 5% |
| 기술적 패턴 3 | 조건 빌더 + AND/OR | 종가 > 50일 이평선 |
| RS 점수 | 임계값 | 80 이상, 90 이상 |
| Market | 체크박스 | KOSPI, KOSDAQ |
| 섹터/테마 | 멀티셀렉트 | sectormap 산업명(대) 기준 |

**기술적 패턴 조건 빌더 구조:**
```
[지표A] [비교연산자] [지표B or 상수] [× 배수(optional)]

지표: 종가, 시가, 고가, 저가, MA(10/20/50/100/200)
연산자: >, <, ≥, ≤, 이격률 이내(%)
```

패턴 1·2·3 간 결합은 AND/OR 선택 가능.

### 4.3 종목 리스트 영역

- sectormap.xlsx의 `산업명(대)` + `산업명(중)` 기준 그룹핑 (기존 파이썬 코드 확인)
- 그룹 내 시가총액 내림차순 정렬
- 그룹 헤더 접기/펼치기(collapse/expand)
- 현재 선택(하이라이트) 종목 표시
- 키보드 ↑↓ 네비게이션 지원
- 표시 정보: 종목명, 코드, 등락률(당일), RS 점수

### 4.4 차트 영역

- **라이브러리**: TradingView Lightweight Charts
- **그리드**: 2×2 / 3×3 토글 (6~9개 동시 표시)
- **차트 내용**: 일봉 캔들스틱 + 이동평균선(10/20/50/100/200) + 거래량
- **메모리 최적화**: 가상화(virtualization) 적용 — 뷰포트에 보이는 차트만 렌더링, 스크롤 아웃 시 인스턴스 파괴
- **페이지네이션**: 종목 순서대로 페이지 단위 표시, `◀ ▶` 또는 종목 리스트 싱크로 전환

### 4.5 스크롤 싱크

| 동작 | 결과 |
|------|------|
| 종목 리스트에서 종목 클릭 | 해당 종목이 포함된 차트 페이지로 이동 + 차트 하이라이트 |
| ↑↓ 키로 종목 이동 | 차트 페이지 자동 전환 (현재 종목이 차트 그리드 첫 번째에 위치) |
| 차트 그리드 페이지 전환 | 종목 리스트 스크롤도 해당 위치로 이동 |

## 6. API 명세

예제:

| Method | Endpoint | 설명 |
|--------|----------|------|
| POST | `/api/db/update` | DB 업데이트 시작 (비동기) |
| GET | `/api/db/status` | 업데이트 진행률 조회 |
| GET | `/api/db/last-updated` | 마지막 업데이트 일시 |
| POST | `/api/screen` | 필터 조건 전달 → 필터링된 종목 리스트 반환 |
| GET | `/api/chart/{code}` | 종목 차트 데이터 (OHLCV + MA) |
| GET | `/api/sectors` | 섹터 목록 조회 |

## 7. 성능 고려사항

| 영역 | 전략 |
|------|------|
